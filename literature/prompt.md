## 背景：AldolRxnMaster 项目概述

**任务定义**：对手性辅基控制的醛醇反应进行 **4-class 立体化学预测**。

- **标签**：`label_joint = Ca × 2 + Cb`，共4类：
  - 0 = RR（Ca=R, Cb=R）
  - 1 = RS（Ca=R, Cb=S）
  - 2 = SR（Ca=S, Cb=R）
  - 3 = SS（Ca=S, Cb=S）
- **Ca**：手性辅基直接控制的立体中心（Evans: 4R/4S 恶唑烷酮决定，预测准确率约89%）
- **Cb**：醛底物控制的立体中心（与醛的 Cahn-Ingold-Prelog 优先级相关，约82.5%）
- **数据集**：共 ~2334 条有效记录（Evans 子集 1654 条，另含 Crimmins/Oppolzer/Myers）
- **已知标签歧义**：class 0（RR）和 class 3（SS）混入了因芳香醛 CIP 优先级翻转导致的"几何-syn 但 CIP-anti"产物，造成内在噪声
- **当前最佳模型**：ExtraTrees，Evans-only TSCV = 0.795 ± 0.052；全数据集 TSCV = 0.624
- **特征体系**：
  - `Morgan` 指纹（醛底物 + 酰亚胺底物，radius=2/3，512/1024 dim）
  - `3D` 描述符（RDKit 3D moments、TPSA、极化率等）
  - 条件特征（碱/路易斯酸/溶剂/温度，one-hot + 数值，~30d）
  - `ald_pri_*`（8d）：醛的 Cb 位 CIP 优先级代理特征（芳香性、α-支化度、杂原子、卤素、环数等）
  - `chiral_*`（7d）：手性辅基特征，含 `chiral_aux_c4_R`（Evans 4R/4S 指示符）
- **技术栈**：scikit-learn（RF/ET/XGB/SVM/LR/kNN/ma_bw 等），time-series CV（TSCV，10-fold），Python 3.10，RDKit
- **核心科学问题**：给定反应底物 SMILES + 实验条件 → 预测两个立体中心的 CIP 绝对构型

### 5. 手性/立体化学处理方式

这是本项目的核心挑战，请重点分析：

1. 论文如何在 **SMILES/图表示** 中编码手性（@/@@ 标注、楔形键、CIP 标签）？
2. 是否证明了标准 SMILES 的手性编码对 Transformer/GNN 是否有效？（参考本调研 [31] 的结论：Transformer 难以从 SMILES 中识别手性）
3. 论文是否区分了 **相对构型（syn/anti）** 与 **绝对构型（R/S）**？本项目的核心混淆源正是 CIP 绝对构型与几何 syn/anti 的不一致。
4. 是否有处理 **CIP 优先级翻转** 导致标签噪声的方法？（本项目中芳香醛 Cb 位的 CIP 翻转导致 class 0/3 标签噪声）
